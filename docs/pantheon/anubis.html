Anubis is the scan and cleanup engine at the heart of Pantheon. With 58 rules across 7 domains, it discovers infrastructure waste that macOS built-in tools, CleanMyMac, and OnyX all miss — hidden caches, orphaned build artifacts, stale AI model weights, and phantom Docker volumes. Anubis now also includes ghost hunting (detecting dead app remnants, formerly Ka) and file deduplication (3-phase SHA-256, formerly Hathor). Every finding includes a category, severity, and safe cleanup recommendation.

Features

🔍58 Scan Rules

Covers 7 domains: macOS caches, Docker, Dev frameworks, AI/ML, IDEs, Cloud infrastructure, and Cloud storage

🛡️Safety First

Every deletion goes to Trash with a SHA-256 decision log. 35 protected paths are hardcoded as undeletable

📊4 Scan Profiles

General, Developer, AI/ML, and Virtualization — scan what matters to you

🧠Neural Classification

Optional brain model classifies files as junk, project, config, model, data, media, archive, or essential

⚡Sub-Second Scans

Rule-based matching with early termination — most scans complete in under 2 seconds

👻Ghost Hunting (ex-Ka)

Detects dead app remnants — startup files, system registrations, and phantom processes left behind by deleted software

🪞File Deduplication (ex-Hathor)

3-phase dedup engine — size grouping, partial hash, full SHA-256. Finds duplicate files eating your storage

🔌MCP Integration

scan_workspace tool available in AI IDEs via MCP server

Usage

$ sirsi weigh # Full workstation scan — discovers all waste
$ sirsi weigh --dev # Developer frameworks only (Node, Go, Rust, Python)
$ sirsi weigh --ai # AI/ML caches only (MLX, CUDA, HuggingFace, Ollama)
$ sirsi weigh --json # Machine-readable JSON output for scripting
$ sirsi judge --dry-run # Preview cleanup — shows what WOULD be deleted
$ sirsi judge --confirm # Execute cleanup — safely moves waste to Trash

How To

Step 1

Install Pantheon — Install via Homebrew or download the binary.

brew tap SirsiMaster/tools && brew install sirsi-pantheon
Step 2

Run your first scan — Scan your entire workstation for infrastructure waste.

sirsi weigh
Step 3

Review findings — Check what Anubis found across all 7 domains. Use --json for scripting.

sirsi weigh --json | jq '.findings | length'
Step 4

Clean with confirmation — Preview what will be deleted, then confirm. Everything goes to Trash first.

sirsi judge --dry-run
sirsi judge --confirm
Step 5

Schedule recurring scans — Set up a weekly scan to keep your system clean.

# Add to crontab or launchd
sirsi weigh --json >> ~/Library/Logs/pantheon-weekly.json

Frequently Asked Questions

Does Anubis work on Linux or Windows?
Anubis runs on macOS and Linux. Windows support is not yet available. The core scan engine is cross-platform Go, but many scan rules target macOS-specific paths (Library, Application Support, etc.).
Will Anubis delete my files without asking?
Never. All destructive operations require explicit --confirm flag. The --dry-run flag lets you preview everything first. Deleted files go to Trash, not permanent deletion.
Does Anubis send data anywhere?
No. Anubis has zero telemetry, zero analytics, and zero phone-home. All scan results stay on your machine. This is a core design principle (Rule A11).
What is the difference between weigh and judge?
weigh scans and reports findings. judge acts on them by cleaning. Always weigh first, then judge --dry-run, then judge --confirm.
Can I run Anubis in CI?
Yes. Use sirsi weigh --json for machine-readable output. Combine with Ma'at for quality gates.
