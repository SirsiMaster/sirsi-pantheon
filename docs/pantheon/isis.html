Isis is the unified health and remedy engine for the Pantheon. She performs health diagnostics (doctor), network security auditing (DNS, WiFi, TLS, firewall, VPN), resource monitoring and throttling (guard), and autonomous remediation (heal). She absorbs the former Sekhmet guardian capabilities into a single, comprehensive health deity.

Features

🩺Health Diagnostics (Doctor)

Comprehensive system health check — disk, memory, swap, load averages, and service status. Identifies degradation before it becomes critical

🔒Network Security Audit

Audits DNS configuration, WiFi security, TLS certificate validity, firewall rules, and VPN status. Flags misconfigurations and vulnerabilities

🛡Resource Monitoring (Guard)

Guards against CPU and RAM hogs. Monitors crash dumps, detects runaway processes, throttles with renice, and warns before crash cascades. Absorbed from Sekhmet

🔧Autonomous Remediation (Heal)

Automatically fixes lint errors, vet warnings, formatting violations, and coverage gaps. Also remediates system health issues discovered by doctor and guard

Usage

$ sirsi isis doctor # Run health diagnostics on the system
$ sirsi isis network # Audit DNS, WiFi, TLS, firewall, and VPN
$ sirsi isis guard # Monitor resources — detect runaway processes and CPU/RAM hogs
$ sirsi isis heal # Run autonomous remediation on current project and system
$ sirsi isis heal --dry-run # Preview fixes without applying them

Frequently Asked Questions

Does Isis work on Linux or Windows?
Yes. Isis is pure Go and runs on all platforms. Network audit and resource monitoring adapt to OS-specific tools automatically.
What does Isis cover?
Four domains: health diagnostics (doctor), network security audit (DNS, WiFi, TLS, firewall, VPN), resource monitoring and throttling (guard), and autonomous code/system remediation (heal). She consolidates what was previously split across Sekhmet and Isis.
Does Isis send data anywhere?
No. All diagnostics and remediation happen locally. Zero telemetry.
Will Isis break my code or system?
Isis applies only safe, deterministic fixes. Use --dry-run to preview all changes before applying. Resource guard throttles with renice before escalating.
Can I run Isis in CI?
Yes. Use sirsi isis heal --auto followed by sirsi maat --fail-under 85 for an automated fix-then-verify pipeline.
