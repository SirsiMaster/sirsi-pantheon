General FAQ

Question Answer
What platforms does Pantheon support? macOS is the primary platform with full support for all features. Linux supports CLI and Go tooling (scan, guard, maat, thoth). Windows has basic CLI support.

Some features are macOS-specific: Apple Notes ingestion (Seshat), ANE/Neural Engine detection (Hapi), Chrome profile path resolution, Spotlight and Launch Services repair (Horus), LaunchAgent management, and crashpad monitoring (Sekhmet).
Does Pantheon send any data externally? No. Zero telemetry, zero analytics, zero phone-home. All scan results, file paths, process names, and audit logs stay on your machine. Pantheon has no network calls, no usage tracking, and no crash reporting. This is enforced by Rule A11 in the project governance and is non-negotiable.
Can I install just one deity instead of the full bundle? Yes. Each deity is independently distributable via Homebrew:

brew install SirsiMaster/tools/pantheon-thoth
brew install SirsiMaster/tools/pantheon-maat
brew install SirsiMaster/tools/pantheon-anubis

Or install the full bundle with brew install sirsi-pantheon to get everything.
How big is the binary? The full Pantheon bundle (all deities) is approximately 12 MB. Individual deity binaries range from 5-8 MB depending on the module. All binaries are statically compiled Go with zero external dependencies.
Is Pantheon open source? Yes. Pantheon is released under the Apache 2.0 license. The full source code is available on GitHub.
Who built this? Sirsi Technologies (sirsi.ai) — an infrastructure intelligence company building tools that help developers and teams understand, optimize, and govern their compute environments.

Deity-Specific FAQ

Question Answer
What's the difference between anubis scan and pantheon scan? They are the same thing. pantheon scan is a convenience shortcut that routes to the Anubis scanner. Both run the same 58 scan rules across the same 7 domains.
Does Thoth work with Claude and Cursor? Yes. Thoth exposes an MCP (Model Context Protocol) server that integrates with any MCP-compatible AI IDE, including Claude Code, Cursor, and Windsurf. Run pantheon mcp to start the server, then configure your IDE to connect.
Can Ma'at run in CI? Yes. Use pantheon maat audit --json for CI/CD pipelines. The JSON output includes the Feather Weight score (0-100), per-package coverage, lint results, and a pass/fail gate. Set a threshold with --min-score 85 for automatic enforcement.
Does Seshat upload anything to NotebookLM automatically? No. Seshat exports files to a local directory and opens the NotebookLM browser page. You manually drag-and-drop the exported files into NotebookLM. No automatic uploads, no API calls, no data leaves your machine without your direct action.
What does Hapi detect? Hapi detects hardware accelerators across four vendors: NVIDIA GPUs (CUDA cores, VRAM), AMD GPUs (ROCm), Apple Silicon (GPU cores, unified memory), and Apple Neural Engine (ANE) / NPU. It reports VRAM allocation, compute capability, and recommends the optimal acceleration path.
Can I use Hathor (dedup) on network drives? Yes. Pass the mount path directly: pantheon mirror /Volumes/NAS-Share. Hathor's 3-phase dedup engine (size grouping, partial hash, full SHA-256) works on any mounted filesystem. For large network volumes, consider using --min-size to skip small files.

Troubleshooting

Problem Solution
pantheon scan is slow Use --quick for fast mode, which skips deep directory walks and heavy rules. You can also exclude large directories:

pantheon scan --quick --exclude node_modules,vendor,.git

A full scan with 58 rules across 7 domains takes 10-30 seconds on most machines. Quick mode finishes in under 5 seconds.
Chrome history can't be read Chrome locks its SQLite database while running, which prevents Seshat from reading history and bookmarks. This is handled automatically — Pantheon copies the database to a temp directory before reading it.

If you still see errors, try closing Chrome first, or check that your Chrome profile path is correct (macOS default: ~/Library/Application Support/Google/Chrome/Default).
Command not found after install Your shell may not have the Homebrew bin directory in its PATH. Try:

brew link sirsi-pantheon

If that doesn't work, verify the binary exists with brew list sirsi-pantheon and ensure /opt/homebrew/bin (Apple Silicon) or /usr/local/bin (Intel) is in your PATH.
