Ka KA v1.1.0

Multi-Layer Ghost Matching Eliminates False Positives

Ka's ghost scanner was flagging WhatsApp, Adobe Acrobat, and CleanMyMac as ghosts. 91 false positives. 6.2 GB of "dead" data that was actually active. A four-layer matching strategy fixed it permanently.

The Problem

Ka v1.0 had a ghost detection blind spot. The scanner was reporting active, in-use applications as uninstalled ghosts and recommending their data for deletion.

Application Ghost Files Reported Ghost Size Reality
WhatsApp 51,915 5.8 GB Active chat data
Adobe Suite 141+ ~350 MB Working app components
CleanMyMac 13 ~50 MB Its own cleaning app was a "ghost"

Root Cause 1: Shallow /Applications Scanning

The app enumerator only checked top-level .app directories. It missed nested app bundles inside WhatsApp.localized/ subdirectories, Adobe Acrobat DC/ vendor folders, and other wrapper directories.

Root Cause 2: Exact-Match-Only Deduplication

Ghost dedup compared residuals to installed apps using only exact bundle ID and exact name matches. It missed sub-processes with different bundle IDs, helper apps with variant names, and vendor-prefixed families.

Users running sirsi ghosts --clean could have deleted active application data including WhatsApp chat history, Adobe working files, and CleanMyMac's own application components.

The Fix: Multi-Layer Matching Strategy

Ka v1.1.0 replaced the single-layer exact match with a four-layer matching cascade. A residual is considered "owned" (not a ghost) if any layer matches.

1

Exact Bundle ID Match

The original v1.0 logic. Fastest and most precise. com.whatsapp.WhatsApp matches com.whatsapp.WhatsApp exactly.

2

Bundle ID Prefix/Family Match

Groups all bundle IDs sharing a vendor prefix. If any com.adobe.* app is installed, all com.adobe.* residuals are attributed to it. Catches helper processes, updaters, and subsidiary apps.

3

Normalized Name Substring Match

After normalizing both names (lowercase, strip punctuation), checks if either name is a substring of the other. WhatsApp matches WhatsAppSMB.

4

Nested Directory Scanning

Recurses one level into /Applications subdirectories, scanning inside .localized/ folders and vendor directories for nested .app bundles.

"The multi-layer matching is permanent architecture, not a one-time fix. Any future app that installs in a subdirectory, spawns helper processes under different names, or uses vendor-prefixed bundle IDs will be correctly attributed to its parent app."
