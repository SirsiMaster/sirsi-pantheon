Getting Started

From zero to a clean, governed, AI-aware workstation in seven steps.

1

Install

macOS (Homebrew)

$ brew tap SirsiMaster/tools && brew install sirsi-pantheon

Linux / From Source

$ go install github.com/SirsiMaster/sirsi-pantheon/cmd/pantheon@latest

Individual Deities

Only need one module? Install deities standalone:

$ brew install SirsiMaster/tools/pantheon-thoth   # AI memory only
$ brew install SirsiMaster/tools/pantheon-maat    # Code quality only
$ brew install SirsiMaster/tools/pantheon-anubis   # Infrastructure hygiene only
2

First Scan

Run a scan to discover infrastructure waste across your machine -- caches, build artifacts, orphaned files, and more.

$ sirsi scan
Example output
Scanning 58 rules across 7 domains... Domain Found Size ───────────────────────────────────── General Mac 12 3.8 GB Dev Frameworks 8 1.2 GB AI/ML 3 890 MB Virtualization 2 2.1 GB ───────────────────────────────────── Total 25 8.0 GB

Want a quick one-shot health check instead? Use doctor:

$ sirsi doctor

This runs scan, ghost detection, and resource checks in a single pass and prints a summary health report.

3

Clean Up

Once you know what waste exists, start cleaning interactively:

$ sirsi scan --clean    # Interactive cleanup -- review each finding before removal

Find Ghost Apps

Detect remnants of applications you already uninstalled -- leftover plists, caches, login items, and support files lurking across 17 macOS locations.

$ sirsi ghosts

Find Duplicate Files

Scan directories for duplicate files using a three-phase hash engine (27x faster than naive hashing).

$ sirsi dedup ~/Documents
4

Set Up AI Memory (Thoth)

Thoth gives AI coding assistants persistent memory across sessions. Instead of re-reading thousands of source lines, the AI reads a compact memory file for instant context -- compressing 22,958 lines to 297 lines of context.

Initialize in a project

$ cd ~/my-project
$ sirsi thoth init

This creates a .thoth/ directory with memory.yaml, journal.md, and an artifacts/ folder.

Sync memory from source

$ sirsi thoth sync

Reads your source files and git history to build a compressed project state. Run this after major changes.

Compact before context compression

$ sirsi thoth compact -s "Refactored auth module, added OAuth2 flow"

Use compact when your AI session is getting long and you want to preserve decisions before context is compressed or the session ends.

5

Code Quality (Ma'at)

Ma'at audits your repository for test coverage, lint errors, vet warnings, and formatting issues -- then grades it on a 0-100 scale.

Audit a repository

$ sirsi maat audit
Example output
Ma'at Governance Assessment ─────────────────────────── Coverage: 87.3% PASS Lint: 0 errors PASS Vet: 0 warnings PASS Formatting: all clean PASS ─────────────────────────── Feather Weight: 94/100

Auto-fix findings

$ sirsi maat heal

Isis (the healing engine) automatically fixes all remediable lint, vet, and formatting issues found by the audit.

6

Knowledge Grafting (Seshat)

Seshat ingests knowledge from external AI platforms and browsers, then exports it into formats your tools understand.

Ingest from Gemini

$ sirsi seshat ingest --source gemini

List Chrome profiles

$ sirsi seshat profiles chrome

Export to NotebookLM

$ sirsi seshat notebooklm --profile SirsiMaster

Seshat supports ingestion from Chrome, Gemini, Claude, Apple Notes, and Google Workspace -- and can export to Thoth memory, GEMINI.md, or NotebookLM.

7

Hardware Detection (Hapi)

Hapi detects your hardware accelerators -- CPUs, GPUs, Apple Neural Engine, and VRAM -- so Pantheon and your AI tools know what compute is available.

$ sirsi hapi detect
Example output
Hardware Accelerators ───────────────────────────────────── CPU: Apple M4 Max (16-core) GPU: Apple M4 Max (40-core) ANE: Apple Neural Engine (16-core) RAM: 128 GB unified VRAM: Shared (128 GB) ───────────────────────────────────── Compute Class: Apple Silicon (ANE-capable)
